## DE-403 [HIGH, post_fix_defect, FxFeatureStorestaleHigh 常時 FIRING]
昨日 DM-302 修正で追加した `observe_feature_staleness` がラベル誤用 (strategy_id
引数に TF 名 "H1" を渡す) + semantics 誤り (age_sec がバー間隔 H1=3600s/H4=14400s
を測定) で固定 60s 閾値の `FxFeatureStorestaleHigh` が H1/H4 戦略 22 件全てで
**起動時点から永久 FIRING**。第2チーム数学的確定。
修正:
  1. metrics.py: signature 変更 (strategy_id, age_sec, *, primary_tf="unknown") +
     fx_feature_staleness_sec gauge ラベルに primary_tf 追加
  2. live_store.py: staleness 再定義
     `staleness_sec = max(0, now - (_last_primary_bar_time + TF_SECONDS[tf]))`
     (バー配信遅延時のみ正値、通常 0)
  3. alerting_rules.yml: TF 別閾値 (M15: >1800s / H1: >7200s / H4: >28800s)
再発防止: tests/unit/test_de403_feature_staleness_semantics.py (9 tests)
ファイル: metrics.py, live_store.py, alerting_rules.yml, test_dm302_*.py (整合更新)

## PA-401 [HIGH, post_fix_defect, AUDIT-0529-003 C-01 同型再発]
昨日 PA-303 修正で main.py の `initialize()` 内 LIVE Gate 拒否時の
`RiskAlertEvent.publish()` が EventBus 未起動 (start() は line 1004) +
EmailNotifier 未 subscribe で silently drop されていた。runner.py 側は
AUDIT-2026-0529-003 C-01 で同等修正済だが main.py に移植漏れ。
修正: idempotent helper `_ensure_eventbus_and_notifier_started()` 追加。
      `_eventbus_started_early` / `_email_notifier_started` フラグで二重 start /
      二重 subscribe を防止。LIVE Gate 拒否 publish 直前で helper を呼ぶ。
再発防止: tests/integration/test_pa401_main_eventbus_startup_order.py (5 tests)
ファイル: main.py

## PA-412 [HIGH, post_fix_defect] + Phase 4.5 追加修正
PLATFORM_ALLOW_MAIN_PRODUCTION_LIVE escape hatch 通過時に PA-401 (Email dead-path) +
PA-103 元因 (既存ポジション無視) が同時顕在化。
修正:
  1. CRITICAL ログ強化 (PA-412 タグ + deprecation 警告 + 二重リスク明示)

  **Phase 4.5 事前 diff 検証で WARNING_BLOCKING 発見**: `_handle_risk_alert`
  dispatcher に `elif kind == "escape_hatch":` 分岐が漏れて silently drop
  していた問題を事前修正。`_EMAIL_THROTTLE_INTERVAL_SEC` に "escape_hatch": 0.0
  を明示登録 (運用意図の明示と将来事故防止)。
再発防止: tests/integration/test_pa412_escape_hatch_warning.py (4 tests)

## RE-MISSED-401 [MEDIUM] get_trade が 5xx/timeout と 404 を混同
第2チーム検証で発見。RE-402 の exit 側ミラー (RE-401/411 等と同じ「例外を None に
握りつぶす」 anti-pattern)。
修正: `TradeNotFoundError` 例外を新設 (BrokerError 派生)。
      `get_trade` の except を transient (5xx/timeout/非404 4xx → BrokerUnavailableError)
      / permanent (404 → TradeNotFoundError) で分離。
      `close_trade` pre_check は `TradeNotFoundError` のみで stale=NOT_FOUND 判定
      (transient は retry 経路へ)。
再発防止: tests/fault/test_re_missed_401_get_trade_exception_discipline.py (8 tests)

## RE-411 [MEDIUM] submit_order TimeoutError 計装漏れ
昨日 DM-302 で `cancel_order` のみに `_record_api_metric` 計装、`submit_order` の
outer except で TimeoutError/5xx の metric が drop。
修正: submit_order outer except 内で `_record_api_metric(status='timeout'
      if isinstance(exc, TimeoutError) else '5xx', ...)` を呼ぶ。
再発防止: tests/unit/test_re411_submit_order_timeout_metric.py (4 tests)
