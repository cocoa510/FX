## PA-501 [HIGH, post_fix_defect] strategies_metadata 参照切断バグ
EmailNotifier.__init__ の `strategies_metadata or {}` で空 dict (falsy) を渡された
場合に右辺で新規 dict 生成 → 呼出元 (main.py の self._strategies_metadata) と
参照切断。helper 経由 (escape hatch / Gate 拒否反復) で initialize 前に
EmailNotifier を生成すると、その時点では戦略ロード前の空 dict を受け取るため
EmailNotifier 側の保持インスタンスが独立し、後段の main.py:691 の populate が
反映されない。escape_hatch / Gate 拒否経路で metadata 永続的に '-' 表示。
修正: `strategies_metadata if strategies_metadata is not None else {}` に変更 (1 行)
再発防止: tests/unit/test_pa501_email_notifier_metadata_reference.py (5 tests)

## RE-501 [HIGH, post_fix_defect] BrokerGateway.get_trade が RE-MISSED-401 修正を gateway 層で無効化
broker_gateway.py:469-477 の `except Exception → return None` が直前修正 (eccab9a)
の TradeNotFoundError / BrokerUnavailableError 両方を握りつぶし、gateway 経由の
本番 2 経路で修正効果ゼロ。特に _recover_stale_trade_pnl で transient 障害が
clear_exposure 誤呼出 → 次バー二重建玉リスク (INC-2026-0526 exit 側経路で再発)。
修正:
  1. broker_gateway.py: 例外を 3 種に細分化
     - TradeNotFoundError (permanent 404) → そのまま raise
     - BrokerUnavailableError (transient 5xx/timeout) → そのまま raise
     - その他 Exception → BrokerUnavailableError でラップして raise (silent drop 廃止)
     - 循環 import 回避は関数内 import
  2. strategy_slot.py:_recover_stale_trade_pnl: transient_skip フラグ導入
     - TradeNotFoundError → exposure クリア + state クリア (permanent stale)
     - BrokerUnavailableError → exposure / state 共にスキップ (次バー再試行に委ねる)
     - INC-2026-0526 同型の幻ポジション (exit 側経路) を構造的に遮断
再発防止: tests/fault/test_re501_broker_gateway_exception_passthrough.py (7 tests)

旧コメント「OANDA open 22:00 は月曜 (day_of_week==1) 扱いとなる」は誤記。
追加修正:
  - alerting_rules.yml コメント L140-156 に既知制約を明記 (誤記削除)
