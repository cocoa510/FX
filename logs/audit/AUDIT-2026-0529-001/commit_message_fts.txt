## RE-302 [HIGH] OANDA タイムアウト時の幻 trade_id 格納 (実損失経路、INC-0526 同型)
broker_gateway.get_open_trades が OANDA 例外を握りつぶし [] を返却していたため、
strategy_slot:792-799 で fill.broker_order_id を state.open_trade_id に格納し、
次回 close_trade が 404 で幻ポジション化する経路を遮断。
BrokerUnavailableError(ConnectionError) 専用例外を導入し例外伝播へ変更。
strategy_slot._resolve_live_trade_id で is_live ガード追加、LIVE + trades=[] は
entry_rollback ルートへ合流。再発防止: test_re302_oanda_timeout_phantom_trade (5 tests)

## RE-301 [HIGH] _resolve_live_trade_id instrument fallback runtime 残存 (INC-0526 同型)
restore_positions (INC-0526 R1) では instrument+direction fallback 廃止+
claimed_trade_ids 排他制御済だが、runtime 経路 strategy_slot:805-823 には未適用
だった問題を修正。同一 instrument・direction 複数 LIVE 戦略で別戦略 trade を
誤 close する実損失経路を遮断。matching=[] 時の instrument 再マッチを撤廃し
entry_rollback へ合流。claimed_trade_ids ランタイム排他制御は次回 audit-loop
で検討と明記。再発防止: test_re301_runtime_instrument_fallback (3 tests)

## DE-303 [HIGH] SharedDataProvider.poll_latest silent failure
data_provider:154-162 の except が logger.exception のみで RiskAlertEvent
未発火 → OANDA candle 障害で全戦略 idle 化しても観測不能だった問題を修正。
_consecutive_poll_failures カウンタ + 30 回連続失敗で RiskAlertEvent
(alert_type=market_data_poll_failure) 発行。one-shot フラグで連投抑止、
成功時リセット。EmailNotifier._RISK_ALERT_KINDS に該当 kind を live_anomaly
集約 (DM-306 と整合)。再発防止: test_de303_poll_latest_failure_alert (6 tests)

## DM-306 [HIGH] OANDA 60秒断絶通知が主要 3 経路 dead (1 行修正で復活)
health_check.py:417 が publish する oanda_connection_lost は _RISK_ALERT_KINDS
未登録 → silently drop されていた問題を修正。Prometheus/Email/Dashboard 全 3
経路で機能しない silent failure を Email 経路で復活。_RISK_ALERT_KINDS に
oanda_connection_lost: live_anomaly を 1 行追加。再発防止:
test_dm306_oanda_connection_lost_email (2 tests)

## DM-302 [HIGH] OANDA API latency / FeatureStore staleness alert 永久 Inactive
metrics.py:562-591 に observe_oanda_api_latency 等 4 メソッド定義済だが本番
呼出 0 件 (grep 確認) → FxOandaApiLatencyHigh / FxFeatureStorestaleHigh が
デッドアラートだった問題を修正。client.py の 5 OANDA API メソッド
(submit_order/cancel_order/get_account_info/get_open_trades/get_candles) に
time.perf_counter() 計測 + observe_oanda_api_latency / inc_oanda_api_status
計装追加。live_store.update_bar に observe_feature_staleness 計装追加。
inc_oanda_api_status の status_code 型を int から int|str に拡張。
再発防止: test_dm302_metrics_instrumentation (7 tests)
